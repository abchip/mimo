Trae le sue fondamenta teoriche nel modello di rappresentazione della conoscenza proposto da M. Minsky, caratterizzato dai concetti di Frame/Slot/Facet.
  • Frame L'elemento oggetto della definizione.
    E' una struttura composta, assimilabile a un Record in Pascal, una Structure in linguaggio C o a una Class in Java
    Implementa ereditarieta' ibrida tramite il legame ako (a-kind-of) e usa (used-as).
    Una sola natura, ma tanti ruoli che possono essere interpretati, e' un approccio, gia' collaudato in linguaggi come Java e garantisce flessibilita' e semplicita' allo stesso tempo.
    Contiene caratteristiche statiche (proprieta') e dinamiche (comportamenti) denominate slot.

    All'interno dei componenti modeling, runtime e persistence sono definiti i frame necessari al framework stesso:
    • modeling frame, slot, facet
    • runtime system, session, engine, service, agent, driver
    • persistence sql-table, sql-field

  • Slot E' una proprieta' o un comportamento del frame.
    Puo' rappresentare informazioni statiche come colore, lunghezza o materiale, ma anche procedurali come distanza, eta' o condizioni di alert.
    Ogni proprieta' del frame puo' essere a sua volta collegata ad un altro frame, eventualmente specificando vincoli sul valore, costruendo quindi una rete di frame a sua volta collegabile ad altre reti.
    Uno slot possiede differenti valori o angolazioni con cui puo' essere rappresentato, chiamati facet.

    All'interno dei componenti modeling e runtime sono definiti gli slot necessari al framework stesso:
    • modeling ako, usa, name
    • runtime isa

  • Facet Un punto di vista sullo slot.
    Una determinata proprieta' del frame puo' essere analizzata attraverso molteplici facet.
    Un facet puo' essere considerato un punto di vista sullo slot, o in altri termini un metaslot.
    All'interno del componente modeling sono definiti i facet:
    • value Il valore della proprieta'
    • container Il contenitore del valore: single, collection, set, map
    • domain Dominio di validita' dei valori
    • constraint I vincoli di validita' dei valori
    • cardinality Il numero di valori
    • default-value Il valore di default assunto
    • main-value Il valore minimo
    • max-value Il valore massimo
    il componente runtime definisce i facet:
    • procedure La procedura da eseguire per ottenere il valore
    il componente proxy definisce i facet:
    • sql-field Il campo della tabella sql contenente il valore
    un componente custom potrebbe definire:
    • expire-value Scadenza del valore
E' costituito da un ambiente unificato che consente allo sviluppatore di astrarsi dal dispositivo di esecuzione.
Solo in fase di deploy vengono decise le componenti di:
  • Persistenza: Il contenitore fisico dei dati.
  • Interfaccia: Interazione con l'agente, sia esso un dispositivo mobile, un sistema desk e un server remoto.
  • Distribuzione: Localizzazione delle componenti.
